### 修改1：`miniapp.html`（移动端）

**1. 加入设备配置信息编辑区域（#configSection）**
- 位置：维修类型字段之后、维修措施之前
- 触发条件：`维修类型` 选 "硬件维修" 时显示；选 "软件维修" 时隐藏
- 包含字段：CPU、内存、硬盘、显卡、操作系统、序列号
- 默认值：从当前资产信息卡预填（Intel i7-12700 / 16GB DDR4 / 512GB SSD / NVIDIA RTX 3060 / Windows 11 专业版 / DEL7090-2024-00156）
- 联动逻辑：修改 `selectRadio()` 函数，当 `#maintainTypeGroup` 点击时判断并切换显示

**2. 维修费用 + 完成日期改为条件显示（#completeFields）**
- 包在 `id="completeFields"` 的 div 中，默认 `display:none`
- 触发条件：`维修状态` 选 "已完成" 时显示；其他状态隐藏
- 联动逻辑：修改 `selectRadio()` 函数，当 `#maintainStatusGroup` 点击时判断并切换显示

**3. `fillMaintainForm()` 设置初始状态**
- 编辑模式：根据 `record.status` 和 `record.repairType` 设置 `#completeFields` 和 `#configSection` 的初始显示/隐藏
- 新增模式：默认状态"待处理"→隐藏费用；默认类型"硬件维修"→显示配置区域

**4. 残留代码清理**
- 删除了 `setRadioGroupValueById('maintainResultGroup', '处理中')` 死代码（上一轮已做，本次确认无残留）

### 修改2：`maintenance.html`（PC端）

**1. 维修状态下拉框加入 onchange 联动**
- `<select id="repairStatus">` 加入 `onchange="onStatusChange()"` 属性

**2. 维修费用 + 完成日期改为条件显示（#completeFields）**
- 包在 `id="completeFields"` 的 div 中，默认 `display:none`
- 触发条件：`维修状态` 选 "已完成" 时显示；其他状态隐藏

**3. 新增 `onStatusChange()` 函数**
- 读取当前 `#repairStatus` 的值
- 判断是否为"已完成"，对应显示/隐藏 `#completeFields`

**4. `loadRecordToDrawer()` 末尾加入 `onStatusChange()` 调用**
- 编辑/查看记录时，根据已保存的 status 正确设置费用/日期字段的初始显示状态

**5. 硬件维修→配置信息区域（已有功能确认）**
- `#configChangeArea` 和 `onRepairTypeChange()` 函数已存在且逻辑正确
- 选择资产后自动根据资产分类（台式电脑/笔记本/打印机等）动态生成对应配置字段
