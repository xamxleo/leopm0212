## 工作习惯

### 交付偏好
- 喜欢**先看到结果**，再提修改意见，而非先讨论方案
- 倾向于**集中反馈**（一次列出多个问题），而非逐条来回
- 对 UI 细节敏感：**菜单一致性、按钮跳转、面包屑导航**都会逐一检查
- 发现 bug 会直接说明现象，不绕弯子

### 沟通风格
- 简洁直接，不需要客套
- 会主动提供截图说明问题（如字典页面截图）
- 任务描述清晰，如"菜单顺序上，将维修管理放在资产码管理上面"

### 项目管理习惯
- 多项目并行（当前是 `2026-05-20-task-6`，后续还有其他项目）
- 每个项目有独立工作目录
- 期望 AI 记住跨会话的上下文和习惯

## 技术偏好

### 前端
- 页面间跳转逻辑要求严格一致（所有 PC 页面左侧菜单必须统一）
- 死链/无效按钮零容忍（如 `href="#"` 必须修正）
- 移动端和 PC 端功能要对齐
- **禁止使用浏览器原生弹窗**（`alert()` / `confirm()` / `prompt()`），WorkBuddy 内置浏览器不支持，必须写自定义 modal 弹窗（动态创建 div + overlay）

### 数据结构
- 偏好**统一配置、集中管理**（如数据字典统一管理所有下拉选项）
- localStorage 作为前端数据持久化方案
