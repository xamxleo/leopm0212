## audio.html 技审流程重构

### 流程变更（按用户说明）
- **技审中** = 自动状态（三方接口异步处理），不展示"技审通过"按钮
  - 技审完成且无错误 → 自动转到**待确认**
  - 技审完成但有错误 → 留在技审中，展示错误，管理员可驳回重生成
- **待确认** = 管理员试听环节，展示播放器，可确认通过或驳回生成

### 具体修改
1. `.ac-status.reviewing` CSS：红色（错误色）→ 蓝色（处理中色），与"生成中"一致
2. `createAudioCard()` — 技审中卡片去掉"技审通过"按钮，只保留"驳回重生成"
3. `createAudioCard()` — 待确认卡片增加 `<audio controls>` 播放器（不含下载按钮，已确认状态才含下载）
4. `confirmAudio()` — 只保留 `pending → confirmed` 逻辑（`reviewing → pending` 已由 `autoTechReview()` 自动处理）
5. `autoTechReview()` — 仅当 `reviewErrors` 为数组且不含 `type: 'error'` 时才自动转 pending（警告不阻塞自动通过）；`reviewErrors` 为 `null` 表示三方接口尚未返回，不自动流转
6. 示例数据调整：`audio-005` 含错误（留在技审中），`audio-006` 仅警告（自动转待确认），`audio-007` 无错误（自动转待确认）
7. AI 助手引导文案更新
