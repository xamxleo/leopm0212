# 广播AI内容生产平台 — 详细模块设计

## 一、项目概述

### 1.1 项目背景
基于AI技术构建广播电台内容生产平台，实现从数据采集、内容生成到节目成品的全流程自动化/半自动化生产，提升内容生产效率与质量。

### 1.2 核心业务流程

```
数据采集 → 文本预处理 → 热点/内容识别 → AI内容生成 → 智能校对 → 语音合成 → 节目打包
                                              ↑           ↑
                                        文字审核API    音频人工审核
```

### 1.3 双轨并行生产模式

| 轨道 | 输入 | 处理流程 | 输出 |
|------|------|----------|------|
| **新闻线** | 热点数据/新闻源 | 热点采集 → 新闻稿件生成 → TTS | 新闻播报音频 |
| **音乐线** | 歌曲信息 | 歌曲串词生成 → TTS | 音乐节目音频 |

## 三、功能模块详细设计

### 3.1 数据采集模块

#### 3.1.1 功能描述
负责从多渠道采集原始内容数据，为后续AI生成提供素材输入。

#### 3.1.2 数据来源

| 数据类型 | 来源渠道 | 采集方式 | 更新频率 |
|----------|----------|----------|----------|
| 热点新闻 | 新闻网站/API/ RSS | 自动爬取/API接入 | 实时/定时 |
| 天气数据 | 天气服务API | API接入 | 每日 |
| 歌曲信息 | 暂不实现（预留接口） | - | - |
| 用户投稿 | 系统上传/邮件 | 手动上传/自动接收 | 实时 |

#### 3.1.3 核心功能

1. **数据源配置**
   - 支持配置多个数据源（URL、API Key、采集规则）
   - 支持RSS订阅源管理
   - 采集频率设置（实时/每小时/每日）

2. **自动采集任务**
   - 定时任务调度（Quartz/Cron）
   - 采集日志记录
   - 异常告警（采集失败通知）

3. **数据清洗（初步）**
   - 去重（基于URL/标题相似度）
   - 格式标准化（HTML→纯文本）
   - 敏感信息过滤

4. **数据存储**
   - 原始数据存入"原始素材库"
   - 保留原始格式+清洗后格式

#### 3.1.4 界面原型参考

参考截图2左侧"待处理中心"风格：
- 数据源配置列表
- 采集任务状态看板
- 今日采集统计

---

### 3.2 文本预处理模块

#### 3.2.1 功能描述
对采集的原始文本进行清洗、分词、结构化提取，为后续AI生成准备标准化输入。

#### 3.2.2 核心功能

1. **文本清洗**
   - 去除HTML标签、特殊字符
   - 去除广告、导航等无关内容
   - 编码统一（UTF-8）

2. **内容结构化**
   - 标题提取
   - 正文提取（基于密度算法或模型）
   - 发布时间/来源提取
   - 关键词提取（TF-IDF/KeyBERT）

3. **文本分块**
   - 长文本自动分段
   - 按语义切割（保持段落完整性）
   - 生成摘要（抽取式/生成式）

4. **标签化**
   - 自动分类（时政/娱乐/体育/科技等）
   - 情感分析（正面/负面/中性）
   - 重要性评分

#### 3.2.3 输出格式

```json
{
  "id": "raw_001",
  "title": "原标题",
  "content": "清洗后正文",
  "summary": "自动摘要",
  "source": "来源网站",
  "publish_time": "2026-05-21T08:00:00Z",
  "category": "科技",
  "keywords": ["AI", "广播", "内容生产"],
  "sentiment": "positive",
  "priority": 8.5,
  "chunks": ["分段1", "分段2"]
}
```

### 3.3 热点/内容识别模块

#### 3.3.1 功能描述
识别新闻热点、提取核心话题、判断内容价值，决定哪些内容值得生成播报。

#### 3.3.2 核心功能

1. **热点识别**
   - 多源聚合：同一事件多源报道聚合
   - 热度计算：基于报道量、传播量、时间衰减
   - 趋势预测：热度上升/下降判断

2. **话题聚类**
   - 相似文章聚类（向量化+聚类算法）
   - 话题命名（基于关键词+LLM生成）
   - 话题关联图谱

3. **内容价值评估**
   - 时效性评分
   - 重要性评分
   - 受众匹配度（与频道定位匹配）

4. **选题推荐**
   - 每日热点Top N推荐
   - 个性化推荐（基于频道历史数据）
   - 人工干预（编辑可调整优先级）

#### 3.3.3 输出

生成"选题池"，供编辑/AI选择：

```json
{
  "topic_id": "topic_001",
  "topic_name": "AI技术突破",
  "articles": ["raw_001", "raw_002"],
  "heat_score": 92.5,
  "trend": "rising",
  "recommended": true,
  "category": "科技",
  "abstract": "话题摘要"
}
```

### 3.4 AI内容生成引擎

#### 3.4.1 功能描述
核心模块，基于LLM生成各类广播稿件（新闻播报稿、歌曲串词、节目导视等）。

#### 3.4.2 生成场景

| 场景 | 输入 | 输出 | 生成策略 |
|------|------|------|----------|
| 热点播报 | 热点话题+多源信息 | 播报文案（200-500字） | 综合多源，生成口语化稿件 |
| 新闻快讯 | 单条新闻 | 简讯稿（50-100字） | 简洁明了，突出核心 |
| 歌曲串词 | 歌曲名+歌手+风格 | 串词稿（30-80字） | 文艺化表达，衔接自然 |
| 节目导视 | 节目信息 | 导视文案 | 吸引听众，预告内容 |
| 早安播报 | 天气+日期+热点 | 综合播报稿 | 结构化模板+动态内容 |

#### 3.4.3 核心功能

1. **Prompt工程管理**
   - 场景化Prompt模板库
   - 变量替换（{{topic}}、{{weather}}等）
   - A/B测试（不同Prompt效果对比）
   - 版本管理

2. **生成策略**
   - 单条生成：基于单个素材
   - 融合生成：基于多个素材综合
   - 模板生成：基于固定模板+动态数据

3. **生成控制**
   - 字数控制（严格限制输出长度）
   - 风格控制（正式/轻松/文艺）
   - 关键词强制包含
   - 敏感词过滤

4. **输出管理**
   - 生成结果保存到"稿件库"
   - 多版本保留
   - 生成耗时/Token消耗记录

#### 3.4.4 Prompt模板示例（热点播报）

```
【角色】你是一位资深广播新闻编辑，擅长撰写口语化、适合广播播报的新闻稿。

【任务】基于以下热点信息，撰写一段广播播报文案。

【热点信息】
标题：{{title}}
摘要：{{summary}}
关键词：{{keywords}}

【要求】
1. 字数控制在300-500字
2. 语言口语化，适合朗读
3. 开头有吸引力，能引起听众兴趣
4. 包含核心事实和关键数据
5. 结尾有简要评论或引导
6. 禁止使用"据了解""据悉"等模糊表述

【输出格式】
直接输出播报文案，不需要额外说明。
```

### 3.5 智能校对模块

#### 3.5.1 功能描述
对AI生成的稿件进行自动校对，识别错误并提出修改建议。

#### 3.5.2 核心功能

1. **基础校对**
   - 错别字检测
   - 标点符号检查
   - 语法错误检测
   - 数字/日期一致性检查

2. **事实核查**
   - 人名/地名/机构名核实
   - 数据一致性（与原文比对）
   - 时间逻辑检查

3. **广播适配性检查**
   - 口语化程度评分
   - 生僻词检测（标注难读词）
   - 句子长度检查（避免过长句）
   - 同音字检查（避免听歧义）

4. **修改建议**
   - 错误高亮标注
   - 修改建议列表
   - 一键采纳/忽略

### 3.6 语音合成（TTS）模块

#### 3.6.1 功能描述
将审核通过的文本稿件转换为音频，支持多种声音模型选择。

#### 3.6.2 核心功能

1. **声音模型管理**
   - 声音模型列表展示（用户自训练模型）
   - 模型信息：名称、音色描述、适用场景、示例音频
   - 模型状态：在线/离线

2. **TTS合成**
   - 文本→音频转换
   - 语速调节（0.5x-2.0x）
   - 音调调节
   - 停顿控制（标点停顿、段落停顿）

3. **分段合成**
   - 长文本自动分段合成
   - 分段音频合并
   - 背景音乐叠加（可选）

4. **音频管理**
   - 合成音频保存到"音频库"
   - 音频波形预览
   - 音频信息：时长、码率、格式

#### 3.6.3 声音模型展示界面

参考用户要求，系统仅展示可选声音：

| 声音名称 | 音色描述 | 适用场景 | 试听 | 状态 |
|----------|----------|----------|------|------|
| 主播-A | 沉稳大气、新闻感强 | 新闻播报 | [播放] | 在线 |
| 主播-B | 亲切温暖、生活化 | 音乐节目 | [播放] | 在线 |
| 主播-C | 年轻活力、快节奏 | 娱乐资讯 | [播放] | 在线 |

#### 3.7.1 功能描述

#### 3.7.2 核心功能

   - 过渡效果（淡入淡出）

3. **输出管理**
   - 导出格式：MP3/WAV
   - 码率选择
   - 元数据写入（节目名、时长、制作时间）

   - 播出时间
   - 时长
   - 内容摘要

### 3.8 审核管理模块

#### 3.8.1 功能描述
对生成的内容和音频进行审核，确保播出安全。

#### 3.8.2 文字审核（自动）

1. **审核流程**
   ```
   稿件生成 → 调用审核API → 返回审核结果 → 判断通过/不通过
   ```

2. **审核维度**
   - 政治敏感
   - 暴恐内容
   - 色情低俗
   - 虚假信息
   - 广告合规

3. **审核结果处理**
   - 通过：进入下一环节
   - 不通过：标记问题类型，退回修改
   - 疑似：人工复核

4. **审核记录**
   - 审核时间
   - 审核结果
   - 问题类型
   - 置信度

#### 3.8.3 音频审核（人工）

1. **审核流程**
   ```
   音频合成 → 提交审核 → 人工在线试听 → 通过/不通过
   ```

2. **审核界面**
   - 音频播放器（波形显示）
   - 播放控制（播放/暂停/快进/快退）
   - 时间轴标注（标记问题点）
   - 审核意见输入

3. **审核状态**
   - 待审核
   - 审核中
   - 已通过
   - 已驳回（含驳回原因）

## 七、技术实现建议

### 7.1 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端 | Vue3 + Element Plus / Ant Design |
| 后端 | Python (FastAPI/Flask) 或 Java (Spring Boot) |
| 数据库 | PostgreSQL + Redis |
| 消息队列 | RabbitMQ / Redis Queue |
| 任务调度 | Celery / XX-Job |
| LLM接入 | OpenAI API / 文心一言 / 通义千问 |
| TTS | 第三方TTS服务 / 自研模型 |
| 文件存储 | MinIO / 阿里云OSS |

### 7.2 关键实现点

1. **异步处理**：内容生成、TTS合成等耗时操作采用异步任务
